\page installer_guide Windows Installer Guide

[TOC]

This guide covers how to set up and build Windows installers for DsQt applications using
[Inno Setup](https://jrsoftware.org/isinfo.php). The installer system uses CMake templates
that are configured at build time, so project-specific values (app name, paths, version)
are automatically populated.

\see settings_guide
\see cmake_user_presets

# Prerequisites

- [Inno Setup 6](https://jrsoftware.org/isdl.php) installed on your machine
- Python 3.x (for downloading DSAppHost and BridgeSync dependencies)
- A GitHub Personal Access Token (PAT) with access to the Unispace365 private repos

# Overview

The installer system consists of these components:

| File | Location | Purpose |
|------|----------|---------|
| `inno.iss.in` | `cmake/` | CMake template — generates the project-specific `.iss` files |
| `apphost.json.in` | `cmake/` | CMake template — generates DSAppHost configuration |
| `make_installer.bat.in` | `cmake/` | CMake template — generates the installer build script |
| `base_install.iss` | `install/` | Shared Inno Setup script included by the generated `.iss` files |
| `apphost_flags` | `install/` | DSAppHost settings file (KEY=VALUE format, read by CMake) |
| `download_apphost.py` | `install/` | Downloads latest DSAppHost from GitHub |
| `download_bridgesync.py` | `install/` | Downloads latest BridgeSync from GitHub |
| `get_deps.bat` | `install/` | Installs Python packages and runs both download scripts |
| `requirements.txt` | `install/` | Python dependencies for the download scripts |

When CMake configures your project, it generates:

- `install/<AppName>.iss` — production installer (auto-start, WER suppression, etc.)
- `install/<AppName>_dev.iss` — development installer (no production flags)
- `install/apphost.json` — DSAppHost configuration with your app name and paths
- `install/make_installer.bat` — installer build script with the correct app name

# Adding Installer Support to an Existing Project

If your project was created before the installer system was added (or without it), follow
these steps to add it.

## Step 1: Copy the template and script files

From the ClonerSource example, copy these files into your project:

```
cmake/inno.iss.in              → <your-project>/cmake/inno.iss.in
cmake/apphost.json.in          → <your-project>/cmake/apphost.json.in
cmake/make_installer.bat.in    → <your-project>/cmake/make_installer.bat.in
install/base_install.iss       → <your-project>/install/base_install.iss
install/apphost_flags          → <your-project>/install/apphost_flags
install/download_apphost.py    → <your-project>/install/download_apphost.py
install/download_bridgesync.py → <your-project>/install/download_bridgesync.py
install/get_deps.bat           → <your-project>/install/get_deps.bat
install/requirements.txt       → <your-project>/install/requirements.txt
```

> **Note:** `make_installer.bat` is not copied — it is generated by CMake with the correct
> app name automatically.

## Step 2: Customize inno.iss.in (optional)

The default flags from ClonerSource will work out of the box. Edit `cmake/inno.iss.in`
only if you need to enable or disable optional features for your project. The flags
section near the top controls what gets included:

```iss
; Enable/disable by commenting or uncommenting these lines:
#define USE_APPHOST        ; Include DSAppHost wrapper
; #define REPLACE_SHELL    ; Replace Windows Explorer (kiosk mode)
#define USE_BRIDGESYNC     ; Include BridgeSync CMS connector
#define SKIP_APP_ICON      ; No desktop icon for main app (apphost only)
```

You can also add project-specific files, registry entries, or run commands in the
sections after the `#include` of `base_install.iss`.

## Step 3: Add CMake configuration to CMakeLists.txt

Add the following blocks to the end of your `CMakeLists.txt` (after the
`install(SCRIPT ${deploy_script})` line):

### DSAppHost configuration

```cmake
# --- DSAppHost configuration ---
# Read settings from install/apphost_flags (KEY=VALUE format)
# Values can still be overridden via cmake -D flags
set(_apphost_flags_file "${CMAKE_CURRENT_SOURCE_DIR}/install/apphost_flags")
if(EXISTS "${_apphost_flags_file}")
    file(STRINGS "${_apphost_flags_file}" _apphost_lines)
    foreach(_line IN LISTS _apphost_lines)
        # Skip comments and blank lines
        if(_line MATCHES "^#" OR _line STREQUAL "")
            continue()
        endif()
        # Parse KEY=VALUE (only set if not already defined via -D)
        if(_line MATCHES "^([A-Z_]+)=(.*)")
            if(NOT DEFINED "${CMAKE_MATCH_1}")
                set("${CMAKE_MATCH_1}" "${CMAKE_MATCH_2}")
            endif()
        endif()
    endforeach()
endif()

configure_file(
    "${CMAKE_CURRENT_SOURCE_DIR}/cmake/apphost.json.in"
    "${CMAKE_CURRENT_SOURCE_DIR}/install/apphost.json"
    @ONLY
)
```

### Installer build script and Inno Setup generation

```cmake
# Installer build script
configure_file(
    "${CMAKE_CURRENT_SOURCE_DIR}/cmake/make_installer.bat.in"
    "${CMAKE_CURRENT_SOURCE_DIR}/install/make_installer.bat"
    @ONLY
)

# --- Inno Setup installer generation ---
if(NOT DEFINED INNO_BASE_INSTALL)
    set(INNO_BASE_INSTALL "${CMAKE_CURRENT_SOURCE_DIR}/install/base_install.iss")
endif()
cmake_path(ABSOLUTE_PATH INNO_BASE_INSTALL NORMALIZE)
set(INNO_BASE_INSTALL "\"${INNO_BASE_INSTALL}\"")

# Production installer
set(INNO_IS_PRODUCTION "#define IS_PRODUCTION")
configure_file(
    "${CMAKE_CURRENT_SOURCE_DIR}/cmake/inno.iss.in"
    "${CMAKE_CURRENT_SOURCE_DIR}/install/${DS_APP_NAME}.iss"
    @ONLY
)

# Dev installer
set(INNO_IS_PRODUCTION "; #define IS_PRODUCTION")
configure_file(
    "${CMAKE_CURRENT_SOURCE_DIR}/cmake/inno.iss.in"
    "${CMAKE_CURRENT_SOURCE_DIR}/install/${DS_APP_NAME}_dev.iss"
    @ONLY
)
```

## Step 4: Update .gitignore

> **Note:** `install/` is probably already in .gitignore. It should be changed to `install/*`

Make sure your `.gitignore` tracks the source files but ignores generated output:

```gitignore
install/*
!install/download_apphost.py
!install/download_bridgesync.py
!install/get_deps.bat
!install/requirements.txt
!install/base_install.iss
!install/apphost_flags
```

> **Note:** Use `install/*` (not `install/`) so that the negation patterns work correctly.

# Building an Installer

## Quick Start

1. **Build and deploy your project first** — the installer packages the DEPLOY directory.
   Build and install your project using Qt Creator or however you normally initiate a build
   in your environment. From the command line it could look like this:
   ```
   cmake --preset ninja-6.10.2
   cmake --build --preset ninja-6.10.2-release
   cmake --install build/ninja-6.10.2 --config Release
   ```
   > Replace `ninja-6.10.2` with the preset that matches your environment.

2. **Run CMake configure** to generate the `.iss` files (this happens automatically during
   the configure step of your build).

3. **Build the installer:**
   ```
   cd install
   make_installer.bat                # Production installer
   make_installer.bat --non-production # Development (NPI) installer
   make_installer.bat -npi             # Development (NPI) installer (short form)
   ```

4. The compiled installer `.exe` will be in `install/build/`.

## What make_installer.bat Does

1. Locates `ISCC.exe` (Inno Setup compiler) — checks PATH, then standard install locations
2. Runs `get_deps.bat` to install Python packages and download DSAppHost + BridgeSync
3. Compiles the `.iss` file with ISCC

## Downloading Dependencies Separately

If you only need to download DSAppHost and BridgeSync without building the installer:

```
cd install
get_deps.bat
```

To clear a stored GitHub PAT and re-enter it:

```
get_deps.bat --reset-token
```

# Production vs Development Installers

| Feature | Production | Development (NPI) |
|---------|-----------|-------------------|
| Output filename | `AppName-v1.0.0.exe` | `AppName-NPI-v1.0.0.exe` |
| Auto-start on boot | Yes | No |
| Windows Error Reporting suppression | Yes | No |
| CMS environment variable (`DS_BASEURL`) | Yes (if `CMS_URL` defined) | No |
| Shell replacement (kiosk) | Yes (if `REPLACE_SHELL` defined) | No |

# DSAppHost Configuration

The `apphost.json.in` template generates the DSAppHost configuration file. Settings are
read from `install/apphost_flags`, a simple KEY=VALUE file:

```
# install/apphost_flags
APPHOST_KIOSK=false
APPHOST_MEMORY_LIMIT=8000
```

Values can also be overridden via CMake `-D` flags (which take priority over the file):

```bash
cmake --preset ninja-6.10.2 -DAPPHOST_KIOSK=false
```

| Variable | Default | Description |
|----------|---------|-------------|
| `APPHOST_PREVENT_SLEEP` | `true` | Prevent Windows from sleeping |
| `APPHOST_KIOSK` | `true` | Run in kiosk mode |
| `APPHOST_RESTART_APPS` | `false` | Restart apps on a schedule |
| `APPHOST_RESTART_APPS_HOUR` | `1` | Hour to restart apps (0-23) |
| `APPHOST_RESTART_APPS_MINUTE` | `0` | Minute to restart apps (0-59) |
| `APPHOST_RESTART_SYSTEM` | `true` | Restart system on a schedule |
| `APPHOST_RESTART_SYSTEM_HOUR` | `1` | Hour to restart system (0-23) |
| `APPHOST_RESTART_SYSTEM_MINUTE` | `0` | Minute to restart system (0-59) |
| `APPHOST_APP_ARGS` | *(empty)* | Command-line arguments for the app |
| `APPHOST_START_DELAY` | `1000` | Delay before starting app (ms) |
| `APPHOST_RESTART_DELAY` | `10000` | Delay before restarting crashed app (ms) |
| `APPHOST_ENABLE_MONITOR` | `true` | Monitor app health and restart if needed |
| `APPHOST_MEMORY_LIMIT` | `10000` | Memory limit in MB |

# Inno Setup Flags Reference

These flags are set in `inno.iss.in` and control what the installer includes:

| Flag | Description |
|------|-------------|
| `USE_APPHOST` | Bundle DSAppHost and install its config to `%USERPROFILE%\Documents\downstream\common\dsapphost\config` |
| `USE_BRIDGESYNC` | Bundle BridgeSync exe from `install/bridgesync/` |
| `REPLACE_SHELL` | Replace Windows Explorer with DSAppHost (requires `IS_PRODUCTION` + `USE_APPHOST`) |
| `SKIP_APP_ICON` | Don't create a desktop shortcut for the main app (useful when using DSAppHost) |
| `CMS_URL` | Set `DS_BASEURL` environment variable (define in `inno.iss.in` if needed) |

# Installed Directory Layout

The installer creates the following structure on the target machine:

```
C:\Program Files\Downstream\
├── <AppName>\                 (per-app directory)
│   ├── bin\
│   │   ├── <appExeName>.exe
│   │   ├── vc_redist.x64.exe
│   │   ├── qt.conf
│   │   ├── settings\          (writable by users)
│   │   ├── data\              (fonts, images, etc.)
│   │   └── *.dll              (Qt, vcpkg, and other runtime DLLs)
│   ├── plugins\               (Qt plugins: imageformats, multimedia, platforms, etc.)
│   ├── qml\                   (QML modules: Dsqt, QtQuick, QtMultimedia, etc.)
│   └── translations\          (Qt translation files)
└── tools\                     (shared across all Downstream apps)
    ├── DSAppHost\             (if USE_APPHOST)
    │   └── DSAppHost.exe
    └── bridgesync\            (if USE_BRIDGESYNC)
        └── bridgesync.exe
```

# Troubleshooting

## ISCC not found

Install [Inno Setup 6](https://jrsoftware.org/isdl.php) or add its install directory
to your system PATH. The script checks `Program Files (x86)\Inno Setup 6` and
`Program Files\Inno Setup 6` automatically.

## GitHub PAT authentication failed

Your token may have expired. Run with `--reset-token` to clear and re-enter:

```
make_installer.bat --reset-token
```

## Missing VC_redist.x64.exe

The installer expects `VC_redist.x64.exe` in the DEPLOY `bin/` directory. Make sure it's
included in your CMake install rules or manually place it in the deploy output before
building the installer.

## .iss files not regenerated

The `.iss` files are generated by CMake's `configure_file()`. If you edit `inno.iss.in`
or change CMake variables, you must re-run CMake configure:

```
cmake --preset ninja-6.10.2
```
