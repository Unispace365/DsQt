3 · Media

How a plain data model becomes the right on-screen element — image, animated image, vector, video, web page or PDF — and how local & network sources both "just work".

Key files

  • Core/qml/ui/DsMediaViewer.qml — the dispatcher (note: in Dsqt.Core)
  • Waffles/qml/ui/DsVideoViewer.qml · DsWebViewer.qml · DsPdfViewer.qml — per-type wrappers

The media model

A viewer's model.media is a JS object describing the resource. Common keys:

"media": {
    "filepath": "%APP%/data/media/sample.pdf",  // local (%VAR% ok) or http(s) URL
    "type":     "pdf",        // image | video | web | pdf | vector | image sequence …
    "width":  612, "height": 792, // natural size (drives aspect-fit sizing)
    "crop": [0,0,1,1]            // optional normalized crop rect
}

DsMediaViewer — the dispatcher

DsTitledMediaViewer.mediaView contains a single DsMediaViewer. It looks at the model's type (or, failing that, the file extension) and loads the matching element into an internal Loader:

function getComponentForContent() {
    let ctype = contentType || media.type || "";
    if (ctype === "video")  return _lazyLoad("_videoComponent", "DsVideoViewer");
    if (ctype === "image")  return isCropped ? imageComponent : animatedImageComponent;
    if (ctype === "web")    return _lazyLoad("_webComponent", "DsWebViewer");
    if (ctype === "pdf")    return _lazyLoad("_pdfComponent", "DsPdfViewer");
    …
    return getComponentForExtension();  // .mp4/.jpg/.gif/.svg/.html/.pdf fallback
}

video (mp4/mov/webm…)DsVideoViewer

Lazy-load workaround

The per-type wrappers live in Dsqt.Waffles but DsMediaViewer is in Dsqt.Core — Core can't statically import Waffles (would be a cycle). So it loads them at runtime via Qt.createComponent("Dsqt.Waffles", "DsVideoViewer"). That's why WebEngine/Pdf can be optional Qt components: if the type isn't built, the lazy load just warns instead of failing the whole module.

Source resolution — local and network

The dispatcher keeps two strings: the raw source (for type/extension detection) and a resolvedSource the actual elements load:

readonly property string source: media && media.filepath ? media.filepath : ""
readonly property url    resolvedSource: source !== "" ? Ds.env.expandUrl(source) : ""

Ds.env.expandUrl() (from Dsqt.Core) is the key: http(s) / qrc / file URLs pass through unchanged, while bare local paths and %VAR% paths (%APP%, %LOCAL%, …) are expanded and turned into proper file:// URLs. So the same code path handles a CMS http URL and a bundled %APP%/data/... file.

Loading state & the spinner

The dispatcher exposes loading; the viewer shows a themed spinner (and a solid surface fill) over the media region while it's true:

onResolvedSourceChanged: loading = (String(resolvedSource).length > 0)
onMediaLoaded:           loading = false
// each element emits mediaLoaded: images on full progress, video on buffered,
// web on LoadSucceeded, pdf when the page is rendered.

The per-type wrappers (ui/)

Each wrapper is thin: it adapts a Qt element to the common interface the dispatcher/controls expect (a source, a mediaLoaded() signal, and renderedWidth/Height/OffsetX/OffsetY for letterbox-aware overlays).

DsVideoViewer (QtMultimedia Video)

Video {
    property bool loading: true
    signal mediaLoaded()
    onBufferProgressChanged: if (loading && bufferProgress >= 1) { loading = false; mediaLoaded() }
    // hold the last frame at end-of-stream instead of clearing to black:
    endOfStreamPolicy: VideoOutput.KeepLastFrame
}

The controls drive it via the standard Video API: position (writable → seek), duration, playbackState, volume/muted, play()/pause(), and loops (set to MediaPlayer.Infinite to loop).

DsWebViewer (WebEngineView)

Two non-obvious things here:

  • WebEngineView's URL property is url, not source — so the wrapper exposes a source and forwards it: onSourceChanged: root.url = source. (Without this the page never loads.)
  • WebEngine's GPU compositing needs a compatible QSG backend — the app runs Direct3D11 (set in Examples/AssetViewer/main.cpp via DsGuiApplication::configureGraphics). On Vulkan the web view renders broken.

The web controls use the built-in canGoBack/canGoForward/goBack()/goForward()/reload().

DsPdfViewer (PdfScrollablePageView)

A self-contained single-page view that centers & fits one page (navigate with the controls). It exposes the page API the controls need and fits robustly:

readonly property int pageCount:   pdfDoc.pageCount
readonly property int currentPage: pdfView.currentPage   // 0-based
function goToPage(n) { pdfView.goToPage(clamp(n)) }

onWidthChanged: pdfView.fitToView()      // refit on resize (e.g. fullscreen)
onStatusChanged: if (status === Image.Ready && !loaded) { loaded = true; fitToView(); mediaLoaded() }
Why key "loaded" off Image.Ready

A PdfDocument loads asynchronously, so at Component.onCompleted the page count is still 0. Initialise the view (fit + go-to-page + emit mediaLoaded) when the page render reaches Image.Ready, not at completion — otherwise the page never fits to the frame and the loading spinner never clears.
