1 · The Stage

DsWaffleStage is the root you put in your scene. It owns the layer of viewers, the live glass "slots", the fullscreen transition, and the selection/z-order logic.

Key file

  • Library/DsQt/Waffles/qml/waffles/DsWaffleStage.qml

Putting a stage on screen

You give the stage your app content via backgroundContent and then ask it to open viewers. Minimal usage (see the demo's MainView.qml for the real thing):

DsWaffleStage {
    id: stage
    anchors.fill: parent

    // Everything below the viewers, and captured by the glass.
    backgroundContent: [
        AnimatedBackground { anchors.fill: parent },
        MyGalleryUI { anchors.fill: parent }
    ]
}

// Open a viewer from anywhere that can see `stage`:
stage.createViewer({
    "model": { "title": "Hello",
               "media": { "filepath": "%APP%/data/x.jpg", "type": "image", "width": 1200, "height": 800 } },
    "matchAspectRatio": true,
    "stage": stage
});

The three layers

Inside the stage, three sibling Items stack up. Order matters:

Layer (id)What it holdsWhy
glassSlotHolderthe live composite "slots"must be in the render tree so its live ShaderEffectSources keep updating; sits at the very back, hidden behind the backdrop
backdropbackgroundContent (alias)your app content; this is the bottom of the glass stack — the deepest viewer samples it directly
topLayer (z:1)viewers, launcher, fullscreen scrimkept above any backdrop content; its onChildrenChanged rebuilds the glass chain
backgroundContent is an alias

property alias backgroundContent: backdrop.data — assigning to it parents your items into the backdrop layer, where they're both shown and captured into the glass.

The viewer lifecycle

Viewers are created from a Component the stage holds, and parented into topLayer. The default viewer type is the demo skin TitledMediaViewer (a ux/ file):

property Component viewer: Component { TitledMediaViewer {} }

function createViewer(viewerProps) { … completeViewer(viewerProps) … }

function completeViewer(viewerProps) {
    let v = viewer.createObject(topLayer, viewerProps);
    if (!("x" in viewerProps) && !("y" in viewerProps))
        centerViewer(v);                         // open centred unless positioned
    v.closeRequested.connect(() => closeViewer(v));
    v.fullscreenRequested.connect((on) => setFullscreen(v, on));
    selectViewer(v);                            // raise + show controls
    v.playEnter();                               // run the open animation
}

So the flow is: create → place → wire signals → select → animate in. viewerProps is just a JS object whose keys are set as properties on the new viewer (model, sizing flags, animation overrides, the stage back-reference, …).

Signals, not direct calls

A viewer never closes or fullscreens itself. Its buttons emit closeRequested() / fullscreenRequested(bool) and the stage performs the action. That keeps the stage the single owner of lifecycle & z-order.

Selection & z-order

One viewer is "selected" at a time. Selecting raises it to the front and tells every viewer whether it's the selected one (which drives whether its controls show):

function selectViewer(v) {
    if (v === _selectedViewer) return;
    _selectedViewer = v;
    for (…topLayer.children…)  child.selected = (child === v);
    if (v) v.z = ++_topZ;        // incrementing z keeps the newest on top
    rebuildGlass();              // re-order the glass chain to match
}

Tapping empty space calls selectViewer(null) (deselect all). Dragging a viewer also selects+raises it. A TapHandler on the stage drives this.

Closing

function closeViewer(v) {
    if (v === _fullscreenViewer) { _hideController(); _scrimShown = false; _fullscreenViewer = null; }
    if (_selectedViewer === v) _selectedViewer = null;
    v.playExit(() => v.destroy());   // animate out, then destroy
}

Destroying the viewer fires topLayer.onChildrenChanged → rebuildGlass(), which rewires the remaining viewers' glass.

Fullscreen orchestration

The stage owns fullscreen because only one viewer may be fullscreen and there's a shared scrim. setFullscreen(v, on) animates the viewer's geometry to a margin-fit of the stage, shows a scrim that blocks the rest of the UI, raises the viewer above the scrim, and shows the fullscreen controller:

function setFullscreen(v, on) {
    if (on) {
        if (_fullscreenViewer) …snap previous one back…  // only one at a time
        _fsRestore = Qt.rect(v.x, v.y, v.viewerWidth, v.viewerHeight); // remember
        _scrimMode = _scrimModeFor(v);    // "blur" if glass on, else "tint"
        v.fullscreen = true; v.z = _scrimZ + 1; _fullscreenViewer = v; _scrimShown = true;
        rebuildGlass();
        _animateViewer(v, …_fullscreenFit(v)…);
        _showController(v);
    } else { …animate back to _fsRestore, hide scrim + controller… }
}
ConceptHow
Scrim style follows the viewer's glass_scrimModeFor() → "blur" when the viewer & stage glass are on, else "tint"; per-viewer override via fullscreenScrim (auto/blur/tint/none)
Tap a margin to exitthe scrim is a MouseArea; tapping it calls setFullscreen(_fullscreenViewer, false)
Controller is per typeproperty Component fullscreenController: Component { FullscreenController {} }; instance created lazily & reused

Details of the scrim + controller are on the Controls and Glass pages.

Settings the stage reads

The stage pulls its defaults from the merged app_settings collection, under the waffles prefix (see theming & settings):

DsSettingsProxy { id: appSettings; target: "app_settings"; prefix: "waffles" }
property real viewerMinWidth:   appSettings.getFloat("viewer.minWidth", 200)
property real viewerImageWidth: appSettings.getFloat("viewer.imageWidth", 800)
property int  viewerEnterAnimation: _animFromString(appSettings.getString("viewer.enterAnimation", "growBounce"))
property int  controlsIdleMs:   appSettings.getInt("controlsIdleSeconds", 4) * 1000
property real fullscreenMargin: appSettings.getFloat("fullscreen.margin", 80)

Viewers default their per-instance properties from these stage values, so you can set a project-wide default in TOML and still override per viewer at creation.
