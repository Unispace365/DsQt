2 · Viewers

A viewer is one draggable "window" on the stage. DsViewer defines the contract every stage-hosted item must satisfy; DsTitledMediaViewer is the real media viewer built on top of it.

Key files

  • waffles/DsViewer.qml — the base contract
  • waffles/media_viewer/DsTitledMediaViewer.qml — the media viewer behaviour
  • ux/TitledMediaViewer.qml — assembles the viewer + its control sets (the skin)

The DsViewer contract

Everything the stage hosts (media viewers, the launcher, future controllers) derives from DsViewer. It carries the properties & signals the stage relies on:

Item {
    enum ViewerType { TitledMediaViewer, Launcher, PresentationController, FullscreenController, Unknown }
    enum Anim       { None, Fade, Grow, GrowBounce, FadeRise }

    property int  viewerType: DsViewer.ViewerType.Unknown
    property int  viewerWidth: 400
    property int  viewerHeight: 300

    // the stage assigns this: a stage-sized item of everything below this viewer.
    property Item backdropSource: null
    // what the stage captures into OTHER viewers' slots (must be glass-free, at 0,0).
    property Item captureItem: dsViewer

    // enter/exit animation config (defaulted by the stage, overridable per viewer).
    property int  enterAnimation: DsViewer.Anim.Fade
    property int  exitAnimation:  DsViewer.Anim.Fade
    property int  animationDuration: 300

    signal closeRequested()
    signal fullscreenRequested(bool wantFullscreen)
    function toggleFullscreen() { fullscreenRequested(!fullscreen) }
    property bool   fullscreen: false
    property string fullscreenScrim: "auto"   // auto | blur | tint | none

    function playEnter() { … }
    function playExit(onDone) { … }
}
MemberWho sets itWho reads it
viewerWidth/Heightthe viewer (sizing logic) / creatorthe viewer's layout; the stage's fullscreen fit
backdropSourcethe stage (rebuildGlass)the viewer's glass, to sample its slice
captureItemthe viewer (overrides to its glass-free layer)the stage, to composite this viewer into others' slots
closeRequested / fullscreenRequestedthe viewer's buttonsthe stage (performs the action)

Enter / exit animations

One reusable ParallelAnimation drives all styles; playEnter()/playExit() set up opacity / scale / a translate offset for the chosen Anim and run it. A separate transform: Translate is used for the rise/sink so it doesn't fight dragging (which writes x/y directly). The stage calls playEnter() after creating a viewer and playExit(cb) before destroying it.

DsTitledMediaViewer — the media viewer

This is where most viewer behaviour lives. Its visual tree is a single contentLayer (the capture target) containing the media region and the edge "holders" that control sets dock into:

DsTitledMediaViewer (DsViewer) └── contentLayer ← captureItem (glass-free; what other viewers' glass samples) ├── mediaGlass DsGlassBackground for the media region (off by default) ├── lock overlay swallows input when contentLocked (so you can drag locked web/pdf) ├── body MouseArea drag + select + "wake" the controls ├── mediaView the DsMediaViewer (image/video/web/pdf) ├── loading fill + spinner └── edge holders: topOuter / topInner / bottomOuter / bottomInner / leftOuter / leftInner / rightOuter / rightInner / background
captureItem = contentLayer

The viewer sets captureItem: contentLayer. The stage composites that (media + controls, but not the viewer's own glass) into the slots other viewers sample — so an upper viewer's glass shows the lower viewers with their chrome, without feedback loops. This is the "compound glass" trick.

Sizing config

How big a viewer opens is declarative. The relevant properties (defaulted from the stage's settings, overridable at creation):

PropertyMeaning
sizeToMediasize the viewer to the media's natural dimensions
matchAspectRatiopreserve the media aspect ratio; base width is imageWidth
imageWidththe target width when fitting to aspect (default from waffles.viewer.imageWidth)
minWidth/maxWidth/minHeight/maxHeightclamp bounds (default from waffles.viewer.*)
function applySizing() {
    let mw = media.width  || viewerWidth,  mh = media.height || viewerHeight;
    let w = sizeToMedia ? mw : (matchAspectRatio ? imageWidth : viewerWidth);
    let h = …;
    if (matchAspectRatio && mw > 0 && mh > 0) { // fit w/h to aspect, then clamp to [min,max] … }
    viewerWidth = Math.round(w); viewerHeight = Math.round(h);
}

Called on Component.onCompleted and whenever a sizing input changes.

Selection & control visibility

The stage sets selected. The viewer maps selection (and fullscreen, and an idle timer) onto which control sets are visible:

property bool selected: false
property bool controlsAwake: true      // reset by interaction; cleared by the idle timer

function _applyControls() {
    let base = selected && !fullscreen;            // outer controls follow this
    let innerOn = base && (controlsAwake || controlsIdleMs <= 0); // inner also idle-hide
    for (…controls…) c.opacity = (_edgeIsInner(c.edge) ? innerOn : base) ? 1 : 0;
}

So outer controls (title, corner buttons) show while selected; inner controls (over the content, e.g. the transport bar) additionally fade after idle and return on interaction. In fullscreen, the viewer hides its own controls and the stage's fullscreen controller carries them. (Full detail on the Controls page.)

Content lock

Interactive media (web/pdf) grabs input, which would block dragging the viewer. The contentLocked flag raises an input-swallowing MouseArea over the media so a locked viewer can be moved/used as a panel; the media controls' lock button toggles it.

How controls get their glass

The viewer hands its control sets a glassContext object — the source slot, the viewer's geometry, the tint/blur/border tokens, and a refresh tick. A control set drops a DsGlassBackground bound to that context and it self-samples the right slice (see Glass & theming). The refresh tick is bumped after layout because the glass origin is computed with mapToItem, which isn't reactive to the controls being re-parented into their edge holders.

The assembly: ux/TitledMediaViewer.qml

DsTitledMediaViewer defines behaviour but ships with no controls. The ux/ assembly is the skin that lists which control sets to mount and wires them to the viewer's signalObject (itself):

DsTitledMediaViewer {
    id: viewer
    controls: [
        TopOuterControls   { signalObject: viewer.signalObject; model: viewer.model
                              DragHandler { target: viewer } },   // drag by the title
        RightOuterControls { signalObject: viewer.signalObject },     // fullscreen + close
        DsMediaControls    { edge: DsControlSet.Edge.BottomInner; offset: 40
                              signalObject: viewer.signalObject }      // per-type transport
    ]
}
Want a different chrome?

Copy ux/TitledMediaViewer.qml, change the controls: list (add/remove control sets, swap edges), register your file, and point DsWaffleStage.viewer at it. The waffles/ behaviour stays untouched.
