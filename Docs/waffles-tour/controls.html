4 · Controls

Control sets are groups of buttons/widgets that dock to a viewer's edges. The same per-type media controls are reused windowed and in the fullscreen controller, and "inner" controls auto-hide when idle.

Key files

  • waffles/media_viewer/DsControlSet.qml — base type for a control group
  • ux/DsMediaControls.qml — per-type transport/nav (video/pdf/web)
  • ux/TopOuterControls.qml · RightOuterControls.qml — title & corner buttons
  • ux/IconButton.qml · DsCtrlIcon.qml · CloseButton.qml · FullscreenButton.qml
  • waffles/DsController.qml — base for fullscreen controllers (contract + frosted glass)
  • ux/FullscreenController.qml — the docked fullscreen bar (media viewers)

DsControlSet — the base

Every control group derives from DsControlSet. It declares the contract the viewer uses to place & drive it:

Item {
    enum Edge { TopInner, TopOuter, LeftInner, LeftOuter, BottomInner, BottomOuter,
                RightInner, RightOuter, Center, CenterBack }
    required property int edge        // where it docks
    property var  config, model           // set by the viewer (from its model)
    property var  glass: null            // the viewer's glassContext (match the chrome)
    property var  signalObject: null     // usually the viewer (buttons call it)
    property real offset                  // gap from the edge
    signal interacted()                  // "user touched me" → resets idle auto-hide

    // control sets start hidden and fade; visible follows opacity so a faded
    // control doesn't keep eating input.
    opacity: 0
    visible: opacity > 0
    Behavior on opacity { NumberAnimation { duration: 200 } }
}

Edges & how a control set is placed

The viewer's setControls() walks its controls list and re-parents each one into an edge "holder" by its edge, binding width/height/offset. The holders sit relative to the media region:

Outer edges sit around the media (not covering it); inner edges sit over the content. That distinction drives idle auto-hide (below).

The buttons

TypeWhat it is
IconButtona Button with a DsGlassBackground and a colourised VectorImage icon; rounds the corners that face "out" based on its edge
FullscreenButton / CloseButtonIconButtons that call signalObject.toggleFullscreen() / signalObject.close() — i.e. they ask the viewer, which asks the stage
DsCtrlIcona flat (no background) centred, colourised icon used inside DsMediaControls; dims when disabled, tints to accent when active
Icons are colourised SVGs

Icons are SVGs drawn through VectorImage + a MultiEffect with colorization: 1.0; colorizationColor: DsTheme.surfaceText — so one white SVG re-tints to the theme (or to accent when active). Icon assets live under %APP%/data/images/waffles/ (window/ for chrome, media/ for transport).

DsMediaControls — per-type controls, windowed & fullscreen

This is the transport/navigation bar. It is a DsControlSet and it dispatches on the media type, showing nothing for images:

mediaTyperow
video / audioplay/pause · scrubber · loop · mute · volume
pdfpage label · prev · page slider · next · lock
webkeyboard (stub) · back · forward · reload · lock
image— (hidden)
DsControlSet {
    id: mc
    property bool chrome: true      // draw own glass pill (windowed) vs none (in FS bar)
    readonly property var    item:  mv ? mv.mediaItem : null
    readonly property string mtype: mv ? mv.mediaType : ""
    visible: hasControls && opacity > 0

    Loader { sourceComponent: mtype==="video" ? videoComp : mtype==="pdf" ? pdfComp
                                            : mtype==="web" ? webComp : null }
    // each row drives `item` directly, e.g. item.play(), item.goToPage(n), item.goBack()
    // and calls mc.interacted() (poke) on every press so the idle timer resets.
}

Because it's bound to signalObject (the viewer) it can reach viewer.mediaViewer (the dispatcher) for mediaItem/mediaType, and viewer.contentLocked for the lock toggle. The lock button flips contentLocked so an interactive web/pdf stops eating input and the viewer can be dragged.

Idle auto-hide (inner controls only)

Controls that sit over the content get in the way, so inner control sets fade out after an idle period and return on interaction. Outer controls (title, corner buttons) do not auto-hide — they show whenever the viewer is selected. The logic lives in the viewer:

property int  controlsIdleMs: stage ? stage.controlsIdleMs : 4000
property bool controlsAwake: true
Timer { id: idleTimer; interval: controlsIdleMs; onTriggered: controlsAwake = false }

function wake() { controlsAwake = true; idleTimer.restart(); _applyControls(); }
// wake() is called from: body press, HoverHandler.onPointChanged, onX/YChanged (drag),
// and each control set's `interacted` signal (connected once at creation).
A no-op property write emits no signal

Writing a property the value it already holds fires no …Changed signal — so don't rely on onXChanged to re-run logic after such a write; call the apply function directly (which is why wake() re-applies visibility itself rather than leaning on onControlsAwakeChanged). Also: faded control sets set visible:false, so a tap "through" them reaches the viewer and wakes them.

Configure the timeout with waffles.controlsIdleSeconds (default 4; 0 disables).

The fullscreen controller

When a viewer goes fullscreen, its own controls hide and the stage shows a FullscreenController — a docked bar carrying the title, the same DsMediaControls, and the window buttons. It is moveable and collapsible, and does not auto-hide (those affordances replace fading):

┌──┐ ┌──────────────────────────────────────────────────────┐ │◧ │ │ Title │ collapse tab on the left │ │ │ ▶ ──────●───────── ↻ 🔊 ──●── ⤢ ✕ │ title + DsMediaControls + window btns └──┘ └──────────────────────────────────────────────────────┘
  • Built on DsController: the controller contract lives in waffles/DsController.qml — the stage drives viewer / shown / collapsed, and the base supplies a frosted glassContext that samples the fullscreen viewer's own content, so the bar & tab are real glass over the media (a solid colour when glass is off), not a flat scrim.
  • Moveable: the controller's origin (0,0) is the centre of the collapse tab, so the tab stays put through collapse/expand; a DragHandler on the tab and one on the bar (any non-interactive part — title, gaps, or an image's empty controls area) move it, clamped to the stage.
  • Collapsible: the left tab toggles collapsed; collapsed shows only the tab and the bar is dropped.
  • Per type: the stage picks the controller component by viewer type (fullscreenController / presentationController); a new look is just another DsController subclass pointed at by the stage. The embedded DsMediaControls runs with chrome:false so it has no pill of its own.
Adding a button to a viewer

Make an IconButton (or reuse one), drop it into a control set (e.g. RightOuterControls), and either call signalObject.something() or emit a signal the viewer/stage handles. Then make sure the control set emits interacted() on press so it participates in idle auto-hide. Full recipe on the recipes page.
