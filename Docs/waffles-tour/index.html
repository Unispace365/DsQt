The Waffles module

A QML library for building touch-table / large-screen "stage" apps: a full-screen surface onto which you open draggable media viewers (image, video, web, pdf) with frosted-glass chrome, per-type controls, and a fullscreen mode.

This tour walks the source top-down — the mental model first, then each layer with the real files, classes, and properties you'll touch. It assumes you know Qt 6 / QML basics (items, properties, bindings, signals). By the end you should be able to add a viewer, add a control, theme the UI, and wire new media into the demo.

The one-sentence model

A stage (DsWaffleStage) owns a layer of viewers; each viewer wraps a piece of media plus edge-mounted control sets; and a shared glass system lets any surface sample & blur whatever sits behind it, themed by DsTheme tokens.

Source layout

The QML is grouped by role. Knowing which folder a file lives in tells you what it is:

Library/DsQt/Waffles/qml/ ├── waffles/ the core runtime types │ ├── DsWaffleStage.qml the stage: owns viewers, glass slots, fullscreen │ ├── DsViewer.qml base type for everything the stage hosts │ ├── DsGlassBackground.qml reusable frosted-glass element │ ├── DsTheme.qml design-token singleton (colors, glass params) │ ├── media_viewer/ │ │ ├── DsTitledMediaViewer.qml the media viewer (glass + media + controls) │ │ └── DsControlSet.qml base type for an edge-mounted control group │ └── menu_launcher/ radial menu / content launcher (DsContentLauncher…) ├── ui/ thin per-type media wrappers │ ├── DsVideoViewer.qml DsWebViewer.qml DsPdfViewer.qml │ └── (DsMediaViewer.qml itself lives in Dsqt.Core: Core/qml/ui/) └── ux/ concrete, swappable chrome (the "skin") ├── TitledMediaViewer.qml assembles a media viewer + its control sets ├── FullscreenController.qml the docked fullscreen control bar ├── DsMediaControls.qml per-type transport/nav controls ├── TopOuterControls.qml RightOuterControls.qml … └── IconButton.qml DsCtrlIcon.qml CloseButton.qml FullscreenButton.qml
waffles vs ui vs ux

waffles/ = the engine (stable runtime contracts). ui/ = thin adapters around Qt media elements. ux/ = the visual chrome you re-skin per brand. When you "design" a viewer you mostly edit ux/; when you change behavior you edit waffles/.

The render stack (what's drawn, bottom to top)

┌─ DsWaffleStage ─────────────────────────────────────────────┐ │ glassSlotHolder live composite "slots" (one per viewer) │ ← never seen directly │ backdrop your app content (animated bg, gallery…) │ ← backgroundContent │ topLayer (z:1) the viewers + launcher + fullscreen scrim │ │ └─ DsTitledMediaViewer │ │ ├─ glass (samples its slot, blurs, tints) │ │ ├─ media (image / video / web / pdf) │ │ └─ control sets at the edges (title, buttons, …) │ └──────────────────────────────────────────────────────────────┘

Each viewer's glass samples a slot — a live texture of everything beneath that viewer in z-order — so a viewer's frosted chrome shows the backdrop and lower viewers, refracted. That chain is the trickiest idea in the codebase; it gets its own page (Glass & theming).
