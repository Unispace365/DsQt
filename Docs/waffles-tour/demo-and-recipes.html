The AssetViewer example is the reference integration. Read it once end-to-end, then use the recipes below for the things you'll actually do.

How AssetViewer is wired

Key files

  • Examples/AssetViewer/main.cpp — graphics backend, engine bootstrap
  • Examples/AssetViewer/Main.qml — window, fit-view, loads the main view
  • Examples/AssetViewer/qml/MainView.qml — the stage + toolbar + galleries
  • Examples/AssetViewer/settings/ — engine.toml, app_settings.toml, waffles_settings.toml
  • Examples/AssetViewer/data/ — bundled media & icons (deployed next to the app)

main.cpp

Sets a WebEngine-compatible render backend, initialises WebEngine, creates the DsQt engine, and loads the QML entry point:

DsGuiApplication::configureGraphics({ .graphicsApi = QSGRendererInterface::Direct3D11, .colorDepth = 8 });
QtWebEngineQuick::initialize();
dsqt::DsQmlApplicationEngine engine;
… query.initialize(); engine.initialize(); …
engine.loadFromModule("AssetViewer", "Main");  // app.mainView from settings
8-bit + D3D11

Use 8-bit colour — 10-bit shifts the tonal/glass colours. And WebEngine's GPU compositing needs D3D11/Metal/OpenGL — not Vulkan — or the web viewer renders broken.

Main.qml — the 4K fit-view

The whole UI is authored on a fixed 4K canvas and scaled to the window by DsFitView. Press S to toggle fit, 0 for 1:1, Space to toggle centring:

DsFitView {
    preferredWidth: 3840; preferredHeight: 2160; fitEnabled: true
    Loader { source: "/qt/qml/AssetViewer/qml/MainView.qml"; anchors.fill: parent }
}

MainView.qml — the stage in use

Creates the DsWaffleStage, fills backgroundContent with the animated background + a toolbar + thumbnail galleries, and opens viewers via small helper functions. The pattern to copy:

function openLocalImage(file) {
    stage.createViewer({
        "model": { "title": file,
                   "media": { "filepath": "%APP%/data/images/gallery/" + file,
                              "type": "image", "width": 1200, "height": 800 } },
        "matchAspectRatio": true,
        "mediaFillMode": Image.PreserveAspectCrop,
        "stage": stage
    });
}

Recipes

Open a viewer

Call stage.createViewer(props). Every key in props becomes a property on the new viewer. The essentials are model (with media) and stage; everything else is optional sizing/animation overrides:

stage.createViewer({
    "model": { "title": "Network video",
               "media": { "filepath": "https://…/clip.mp4", "type": "video", "width": 640, "height": 360 } },
    "matchAspectRatio": true,
    "enterAnimation": DsViewer.Anim.FadeRise,   // override the settings default
    "stage": stage
});

Local vs network is automatic — a %APP%/… path and an http(s) URL both work (see Media → source resolution). Omit x/y to open centred.

Add a button to a viewer

  1. Make (or reuse) an IconButton that signals intent: 
    // ux/PinButton.qml
IconButton {
    iconSource: "file:///%APP%/data/images/waffles/window/lock.svg"
    onPressed: signalObject?.pinToggled?.()
}
  2. Drop it into a control set (e.g. RightOuterControls) so it docks at an edge.
  3. Handle the call on the viewer (its signalObject), or add a signal the stage handles — mirror how closeRequested/fullscreenRequested flow up to the stage.
  4. Emit interacted() from the control set on press so the button keeps the controls awake.
  5. Register the file in QML_FILES (see below).

Add a new media type

  1. Write a thin wrapper in Waffles/qml/ui/ exposing source, a mediaLoaded() signal, and renderedWidth/Height/OffsetX/OffsetY.
  2. Teach the dispatcher about it in Core/qml/ui/DsMediaViewer.qml — add a branch in getComponentForContent() / getComponentForExtension() that _lazyLoad("_xComponent", "DsXViewer"), and extend mediaType.
  3. If it has controls, add a row to DsMediaControls for that mtype.
  4. Register both files in QML_FILES.

Re-skin the viewer (new chrome)

Copy ux/TitledMediaViewer.qml, edit its controls: list (different control sets, edges, offsets), register it, and point the stage at it:

DsWaffleStage { viewer: Component { MyTitledMediaViewer {} } }

The waffles/ behaviour (sizing, glass, fullscreen, idle) is untouched.

White-label the theme

Re-point DsTheme roles/primitives once at startup — the palette and glass re-derive, no call sites change:

DsTheme.tonal0 = "#0E1A14";   // new surface
DsTheme.accent = "#22C38A";   // new brand hue

Change behaviour via settings (no code)

Edit settings/waffles_settings.toml — viewer bounds, default open width, enter/exit animation names, idle timeout, fullscreen scrim. They flow into the stage and become the per-viewer defaults.

Register a new QML file

Anything new under Waffles/qml/ must be listed in Library/DsQt/Waffles/CMakeLists.txt under QML_FILES (singletons also need set_source_files_properties(... QT_QML_SINGLETON_TYPE TRUE)). Forgetting this is the #1 "my type isn't found" cause.

Working notes

  • Validate QML fast without a full build with qmllint against the built module + Qt import paths. After adding a type/property you'll see stale "not found" warnings until the module is rebuilt — those clear on rebuild; real syntax errors show as Error:.
  • Deploy: settings/ and data/ are installed next to the app (install(DIRECTORY …)). New media/icons/settings need a re-run of install/deploy, or the app won't find them (e.g. a PDF that never reaches Ready).
  • Binary assets are tracked with Git LFS (*.png/.jpg/.svg/.mp4/.gif/.pdf… in .gitattributes) — new asset extensions should be added there too.
  • Optional Qt bits: web needs WebEngineQuick, pdf needs Pdf. If a component isn't built, that media type lazy-loads to a warning rather than breaking the module.
Still open in the codebase

A couple of intentionally-unfinished spots you may meet: the web controls' keyboard button is a stub (on-screen keyboard TODO), and metadata-driven auto-loop (media.loop → loop on open) isn't wired yet (the loop toggle and hold-last-frame are). The menu_launcher/ radial menu is also a work in progress.
